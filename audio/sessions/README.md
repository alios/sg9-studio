# Audio Sessions Directory

This directory contains Ardour session templates and organizational structures for SG9 Studio broadcast workflows.

## Directory Structure

```
audio/sessions/
├── README.md                    # This file
├── snapshots/                   # Session snapshots for version control
├── exports/                     # Exported master files
└── [template files]             # .ardour template files (when created)
```

## Ardour Template Storage

**System Templates:** `~/.config/ardour8/templates/`  
Templates stored in the user configuration directory are available in Ardour's GUI (Window → Templates).

**Repository Templates:** `audio/sessions/`  
Templates in this repository are:
- Version controlled (Git)
- Portable across systems
- Documented with setup instructions

**Recommended Workflow:**
1. Create/modify templates in Ardour GUI
2. Export template via Session → Save Template
3. Copy `.ardour` template file from `~/.config/ardour8/templates/` to this directory
4. Commit to version control
5. On other systems: import by copying to `~/.config/ardour8/templates/`

Alternatively, create a symbolic link:
```bash
ln -s /Users/alios/src/sg9-studio/audio/sessions/SG9-Studio-Broadcast-v1.0.template \
      ~/.config/ardour8/templates/SG9-Studio-Broadcast-v1.0.template
```

## Template Naming Convention

**Format:** `SG9-Studio-[Purpose]-v[Major].[Minor].template`

**Examples:**
- `SG9-Studio-Broadcast-v1.0.template` - Primary broadcast template
- `SG9-Studio-Podcast-v1.0.template` - Podcast-specific template
- `SG9-Studio-Music-v1.0.template` - Music production template

**Versioning:**
- **Major version (v1.0 → v2.0):** Significant changes (track layout, bus structure, workflow changes)
- **Minor version (v1.0 → v1.1):** Minor updates (plugin adjustments, VCA tweaks, added tracks)

## Session Management

### Snapshots

Use Ardour's snapshot feature for session versioning:
- `Snapshot (& keep working on current version)` - Branch session
- `Snapshot (& switch to new version)` - Save and switch

**Storage:** `snapshots/` directory (for documentation, not Ardour session files)

**Naming:** `YYYY-MM-DD_[session-name]_[descriptor].snap`  
Example: `2026-01-19_interview-john-doe_pre-editing.snap`

### Exports

**Storage:** `exports/` directory

**Naming:** `YYYY-MM-DD_[content-title]_[version]_[format].ext`  
Examples:
- `2026-01-19_episode-001_final_stereo.wav` (48kHz/24-bit master)
- `2026-01-19_episode-001_final_podcast.mp3` (Apple Podcasts -16 LUFS)
- `2026-01-19_episode-001_v2_review.mp3` (draft for review)

**Format Standards:**
- **Master:** 48kHz/24-bit WAV (stereo)
- **Apple Podcasts:** MP3 or AAC, -16 LUFS, TP ≤-1.0 dBTP
- **Spotify:** MP3 or OGG, -14 LUFS, TP ≤-1.0 dBTP
- **YouTube:** WAV or MP3, -14 LUFS, TP ≤-1.0 dBTP

## Template Contents

Each SG9 Studio template should include:

**Track Layout** (from [ARDOUR-SETUP.md](../../docs/ARDOUR-SETUP.md)):
- Track 1-2: Host Mic (DSP + Raw safety)
- Track 3-4: Guest Mic (DSP + Raw safety)
- Track 5-6: Aux Input (phone/tablet)
- Track 7-8: Remote Guest (VoIP input + loopback)
- Track 9: Music Bed (stereo file playback)
- Track 10: Cues (Ardour clips/cue triggers)

**Bus Structure:**
- Master Bus (stereo output)
- Mix-Minus Bus (remote guest send, excludes their own voice)
- Music Bus (music submix)
- FX Bus (reverb, delay, etc.)

**VCA Masters:**
- VCA 1: All Host tracks
- VCA 2: All Guest tracks
- VCA 3: Music/SFX tracks

**Plugin Processing Chain** (per [STUDIO.md](../../docs/STUDIO.md)):
1. HPF (High-Pass Filter, 80-100 Hz)
2. Gate (noise gate, -50 to -40 dB threshold)
3. De-esser (LSP Compressor Sidechain mode or Calf)
4. EQ (parametric, 3-5 bands)
5. Compressor (3:1 to 4:1 ratio, -20 to -15 dB threshold)
6. Limiter (master bus only, -1.0 dBTP ceiling)

## Session Backup Strategy

**Backup Frequency:**
- **During recording:** Ardour auto-saves every 60 seconds (configurable)
- **Post-session:** Manual backup to external storage
- **Project milestones:** Git commit with descriptive message

**Backup Locations:**
1. **Local:** `~/.config/ardour8/backup/` (Ardour auto-backup)
2. **Repository:** Git version control (templates, scripts, documentation)
3. **External:** Time Machine, Backblaze, or dedicated backup drive (session files)

**What to Backup:**
- Session directory (`.ardour` file + `interchange/` subdirectory with audio)
- Exported masters (`exports/` directory)
- Session snapshots (if using snapshot workflow)
- Templates (this directory, already in Git)

**What NOT to Backup:**
- `peaks/` directory (regenerated by Ardour)
- `dead/` directory (deleted audio regions)
- Temporary analysis files
